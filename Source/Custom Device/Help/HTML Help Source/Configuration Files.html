Configuration Files

Two configuration files are required to run the Ballard ARINC 429 custom device; the hardware configuration file and the parameter configuration file. The following table describes these files.

File Name Description
Hardware Configuration File

Configures the avionics communications hardware and to deploy the system definition to a real-time target. The schema is specific to Astronics Ballard devices.

Use tools such as BTIXMLEdit or database import utilities to generate the hardware configuration file.

Schema information is installed by the driver API at the following location: C:\Program Files (x86)\Astronics Ballard Technology LabVIEW Driver\Help\index.html.
Parameter Configuration File

Configures the custom device to encode and decode parameters. The file is required for Ballard ARINC 429 system definitions created in System Explorer.

The parameter configuration file must agree with the hardware configuration file. This includes matching labels on each channel and channels configured on the device core.

For more information on the parameter configuration file schema, refer to the Ballard ARINC 429 Custom Device GitHub repository.

This section has the following configuration settings:

  • Hardware—Specifies the path to the hardware configuration file.
  • Parameters—Specifies the path to the parameters configuration file.
  • Refresh—Invokes the Refresh dialog box. Use this dialog box to view changes to a configuration when importing a new parameters configuration file or when the parameters configuration file has changed on disk.
